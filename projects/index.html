    Projects

    Jelly Splash (Android - Google Play/Kindle)

    Jelly Splash is a cool puzzle game for mobile devices where you squash things. link

    This project was a port of an existing iOS game to the android platform. Working as an engineer on this project, we were able to bring this game to the Play Store in under 4 months.

    Notes

    • Apportable / Objective-C
    • Also available on the Kindle App Store
    • Over 5 Million installs on Google Play

    Pocket Village (iOS)

    Pocket Village was Wooga’s first mobile only game. You build villages from your pocket, and exploit the avarice of a balloon trader to gain wealth! link

    As one of the main engineers on this project, I helped develop this game from planning through to implementation. We developed a rudimentary entity/component system to drive the logic of the application, and the UI flow was driven by heavily customised UIKit controllers. I developed some extensive tool support in Ruby, including a series of command line scripts to automate common tasks and a Sinatra based webapp to enable customer support / user account admin.

    Notes

    • iOS / Objective-C
    • Cocos2D
    • UIKit
    • Fun Fact: Pocket Village was No 1. top grossing app in the Mali App Store for a period of time

    Duck Hunter (iOS)

    Duck Hunter was an internal project used at trade shows to demonstrate connectivity between our social network product and various other platforms.

    Duck Hunter was a minigame based application, in the style of Point Blank or WarioWare. I developed a custom OAuth2 client, supporting user connectivity with a social network product we developed at HuzuTech. In addition I worked on a few of the minigames, including an accelerometer based duck herding game.

    Notes

    • UIKit
    • Sparrow (Game Framework)
    • Ruby Tooling

    Small Potatoes Avatar Builder (Facebook)

    This is an avatar builder for a children’s television program featuring singing potatoes (seriously, click here). The app is available on facebook and lets you directly display your creations to your friends. link

    This app was written primarily using the canvas api to composite it’s images. This was at a time when using Flash on iOS platforms was a big concern, so we developed this approach.

    Notes

    • CoffeeScript
    • RequireJS
    • Heroku
    • Ruby / Sinatra

    Horrible Histories Virtual World (Flash)

    The Horrible Histories Virtual World project was a Flash based MMO to tie in with the well known UK children’s book series.

    This was a full stack project, as we developed the entire client / server and infrastructure ourselves. I was involved in all aspects of development, including creating server plugins and shipping client features. We were able to deliver an incredibly ambitious project due to our focus on code quality and testability.

    Tech

    • AS3
    • Python
    • Amazon Web Services
